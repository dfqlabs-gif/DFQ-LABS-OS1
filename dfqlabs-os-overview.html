Overview

What is DFQ Labs OS?

DFQ Labs OS is the internal CRM and sales orchestration platform that runs every stage of the agency's client acquisition pipeline — from first DM to closed deal.

  • Outreach tracking — every DM, reply, and follow-up logged
  • AI at every step — generates, reviews, and QA-filters messages
  • CEO command layer — full-agency health visible at a glance
  • Team accountability — per-specialist daily activity dashboards
  • Revenue forecasting — guaranteed, likely, and weighted pipeline

AI Engine

7-Step Reasoning Pipeline

Step 1
Context Reader
Reads the full DM thread, notes, and CRM history to understand the relationship.
Step 2
Timeline Builder
Maps every touchpoint chronologically and flags gaps, silences, or stage regressions.
Step 3
Stage Detector
Determines the exact pipeline stage and rejects stage-skipping or repeat asks.
Step 4
Objective Planner
Defines the single correct next objective — no multi-tasking a message.
Step 5
Strategy Generator
Picks the psychological strategy and tone for this prospect's archetype.
Step 6
DM Writer
Writes 2–4 sentences. No emojis, no exclamations, no AI buzzwords.
Step 7
Quality Checker
3-stage QA gate: review → adjust → validate. Score 0–100. Auto-adjusts if below 75.
Model
Gemini Flash
All generation routed through Gemini with server-side model fallback chain.

AI Capabilities

AI Coach & AskAI Assistant

AI Coach — Lead Modes
Value DM Full 7-step pipeline output
Follow-up DM Picks up exactly where thread ended
Closing DM Close or Beta Program pitch
Rebuttals Objection-handling playbook
Every output runs through the 3-stage QA gate before display. Score shown as a badge. Auto-adjusts when <75.
2-Step DM Flow
Step 1: AI reads the thread and summarises where the prospect is — the specialist confirms or rejects the read.

Step 2: AI drafts the message. QA gate runs automatically. Adjusted version shown if needed.
AskAI — Chat Bubble
Draggable, always-on chat assistant. @mention any lead by name to give the AI full CRM context for that prospect. Generates DMs inline — all subject to the same QA pipeline. Markdown stripped from all AI output across the entire system.
Quality Gate (3 Stages)
Stage 2 — Review: Scores the draft 0–100 against lead context
Stage 2b — Adjust: Rewrites if score <75, fixing all identified problems
Stage 3 — Validate: Binary approve/reject with scored evidence

Technical Architecture

How It's Built

Frontend
React + TypeScript
Vite dev server. Modular component architecture across 10+ specialist files. Dark/light mode.
Backend
Express + Node.js
REST API for AI calls, data persistence, and session management. Runs on port 5000.
AI Layer
Google Gemini
gemini-3.1-flash-lite. All prompts routed through a shared aiEngine module with server-side fallback chain.
Data Storage
JSON + LocalStorage
Leads persisted via server API with JSON export/import. Browser session auth with idle timeout.
Deployment
Replit
Hosted on Replit with managed workflow. GitHub-synced. Environment secrets managed securely.
Key Modules
aiEngine.ts · aiQA.ts · prompts.ts
constants.tsx · types.ts
CEOTab · TeamTab · AICoach
AskAI · LeadModal · AIQAPanel
Auth System
Role-based access: Founder (full) and Sa'adatu (own leads only). Password-protected portals. Session expires after idle timeout.
Anti-Patterns
No markdown in output. No stage-skipping. No generic AI advice. No double-outreach to same company. QA blocks low-quality DMs.
